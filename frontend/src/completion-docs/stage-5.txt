1. SCROLL ANIMATION SYSTEM
=====================================

  Architecture: useScroll + useTransform (FULLY REVERSIBLE)
  
  ❌ REMOVED: whileInView, viewport={{ once: true }}
  ✅ IMPLEMENTED: scroll-progress-driven transforms
  
  Hook: src/hooks/useScrollAnimation.js
    - Reusable across all sections
    - Returns: ref, opacity, y, scale, scrollYProgress
    - Uses target-based scroll tracking
    - offset: ["start end", "end start"]
    - Default range: [0, 0.4]

=====================================
3. REVERSIBLE ANIMATIONS ✅
=====================================

  ALL animations are driven by useTransform:
  - Scroll DOWN → section appears (opacity 0→1, y 80→0)
  - Scroll UP → section disappears (opacity 1→0, y 0→80)
  - No "once: true" — animations replay on every scroll pass
  - Smooth, continuous interpolation (not keyframe-based)

=====================================
4. HOOK CREATED
=====================================

  File: src/hooks/useScrollAnimation.js
  
  Parameters:
    range (Array): scroll progress mapping [start, end]
  
  Returns:
    ref: attach to section element
    opacity: 0→1 mapped to scroll range
    y: 80→0 mapped to scroll range
    scale: 0.95→1 mapped to scroll range
    scrollYProgress: raw progress value

=====================================
5. STAGGER ANIMATION
=====================================

  SolutionSection & CoursesSection use MANUAL stagger:
  - Each card has individually offset useTransform ranges
  - Cards reveal sequentially as scroll progresses
  - Stagger gap: ~0.05 scroll progress per card
  - Fully reversible — cards hide in reverse order on scroll up
