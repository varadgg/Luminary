2. SHARED COMPONENTS CREATED
================================================================

a) Navbar (src/components/shared/Navbar.jsx)
   - Accepts prop: activePage ("home" | "courses")
   - Renders Home variant when activePage="home"
     → Logo: "LUMINARY" (uppercase, Plus Jakarta Sans)
     → Links: Home (active), Courses, Benefits, About
     → Button: "Register Now"
   - Renders Courses variant when activePage="courses"
     → Logo: "Luminary" (title case)
     → Links: Home, Registration, Courses (active), Benefits, About
     → Button: "Start Learning"
   - Active link is highlighted via conditional className

b) Footer (src/components/shared/Footer.jsx)
   - Accepts prop: activePage ("home" | "courses")
   - Renders Home variant: multi-column layout with
     LUMINARY branding, Education, Platform, Connect sections
   - Renders Courses variant: single-row layout with
     copyright and links (Privacy, Terms, Help, Contact)


3. HOW NAVBAR DUPLICATION WAS RESOLVED
================================================================

Before restructuring, there were TWO separate Navbar files:
  - HomeNavbar.jsx (for Home page)
  - Navbar.jsx (for Courses page)

Resolution:
  1. Created ONE shared Navbar component at
     src/components/shared/Navbar.jsx
  2. Uses `activePage` prop to determine:
     - Which visual variant to render (Home vs Courses design)
     - Which navigation link gets the active state highlight
  3. Deleted both old files:
     - src/components/HomeNavbar.jsx (removed)
     - src/components/Navbar.jsx (removed)
  4. Both pages now import from the same shared component

Same approach applied to Footer:
  - HomeFooter.jsx and Footer.jsx merged into
    src/components/shared/Footer.jsx
